JSX

Como no caso dos componentes, o JSX será tratado com mais detalhes mais tarde, mas por ser um pouco confuso no início vamos apresentá-lo brevemente.

O JSX é a sintaxe de templating do React. Um conceito importante no React é que, ao contrário do Vue ou Angular, tudo (com exceção de CSS) é tratado como Javascript. Isso inclui o template visual: nada é feito usando HTML. Em vez disso se usa o JSX: um “açúcar sintático” que permite que escrevamos código JS quase como HTML.

Exemplificando, em React escrever:

    <div>
    <p>Lorem Ipsum</p>
    </div>

é equivalente a escrever:

    let div = document.createElement("div"),
        p = document.createElement("p");
    
    p.innerHTML = "Lorem Ipsum";
    div.append(p);

A diferença é que, por "traduzir" o código automaticamente, o React nos permite escrever nossas páginas e componentes de maneira mais próxima ao que estamos acostumados: com HTML puro. No entanto, não se engane: a notação JSX possui algumas diferenças cruciais do HTML. Agora vamos ver alguns exemplos mais práticos de como trabalhamos com essa sintaxe para criar nossos elementos no dia a dia e, assim, fixar bem como ele deve ser abordado e no que precisamos prestar atenção. Todos os exemplos podem ser seguidos se forem seguidos também os passos de instalação do react-create-app.
