Listas

Do mesmo modo como não temos diretivas para condicionais, também não temos recursos para diretamente iterar elementos em um array. O que fazer, então? Aqui podemos usar o mesmo recurso que acabamos de usar acima e, dentro do interpolador, forçar a iteração de um array para gerar listas de dados!

Como não podemos também (por enquanto) estruturas de repetição padrões (limitações da ferramenta), devemos usar métodos disponíveis pela própria interface de arrays do Javascript, como a função map.

Assim, se temos um array: var frutasFavoritas = ['Abacaxi', 'Tangerina', 'Pêssego']; e queremos exibir seus elementos em uma lista, podemos abrir o interpolador e dentro dele executar algo assim:

    

        <h1>Minhas frutas favoritas (atualmente)!</h1>
        <ul>
            {frutasFavoritas.map((fruta,index)=>{
            return <li key={index}>{fruta}</li>
        })
            }
        </ul>

Explicando melhor o código acima: o método map recebe uma função que deve ser executada em todos os itens de um array (no caso, frutasFavoritas ). Como no fim das contas o que nos importa é que, para cada item, seja retornado um item (ou seja, um

  • ) na lista, essa função simplesmente retorna o código JSX referente ao elemento que queremos criar.

    Repare que essa função recebe dois parâmetros: fruta e index. Aqui, o primeiro é o conteúdo do array na posição que está sendo trabalhada pelo map, enquanto o segundo é a posição em si. Assim, dentro da "tag" colocamos o valor do array (a fruta).

    Um detalhe importante: no React é bom sempre definirmos um identificador em elementos que são gerados por iteração. Em vez de usarmos o tradicional id aqui usamos key, um atributo especial do React usado para controlar o estado da lista (vamos ver mais sobre isso mais tarde). E atenção: é muito importante esses elementos terem valores únicos. Aqui usamos o próprio index, mas use um padrão de nomeação que você saiba que sempre será único. Assim, evitamos problemas com a aplicação futuramente.
