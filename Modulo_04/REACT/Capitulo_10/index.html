Condicionais

Lembra que para o JSX tudo é convertido em JavaScript? Isso tem um efeito colateral: não separamos o template em arquivos HTML de verdade e, assim, várias “regalias” como diretivas por exemplo ficam inacessíveis. Essa é uma das barganhas que o React faz: em troca de ter que manipular menos o DOM (já que é tudo JS, lembra?) e assim ter melhor performance, abre mão de certos recursos de qualidade de vida. Mas calma, nem tudo está perdido.

Um dos truques do React é que, como aquele é basicamente um meio de executar Javascript dentro do template, ele consegue também interpretar JSX. Isso quer dizer que essa notação:

    {
      <a
        className="App-link"
        href="https://reactjs.org"
        target="_blank"
        rel="noopener noreferrer"
      >
        Learn React
      </a>
    }

é perfeitamente aceitável, o que seria impensável em outros frameworks. Além disso, por ser JS, podemos também atribuir código JSX a variáveis, como por exemplo:

    var link = (
      <a
        className="App-link"
        href="https://reactjs.org"
        target="_blank"
        rel="noopener noreferrer"
      >
        Learn React
      </a>
    );

Com essas ferramentas podemos criar condicionais, mesmo que de maneira um pouco diferente e menos direta do que o Vue ou o Angular provém. Dentro do interpolador em si não podemos usar a expressão if diretamente para comparações (pelo menos não ainda) mas podemos usar operadores ternários padrão do JS. Assim, um modo simples de escrever um condicional seria:

    var flag = Math.random > 0.5; //Aleatoriamente é falso ou verdadeiro
    return (
      {flag ?
        <a
          className=“App-link”
          href=“https://reactjs.org”
          target=“_blank”
          rel=“noopener noreferrer”
        >
          Learn React
        </a>
      : null
    });

Para componentes e apps menores esse jeito é bom o suficiente. Entretanto, à medida em que o projeto, e assim o código, cresce, com essa maneira fica mais difícil ler e manter o mesmo, pois a notação gera muita poluição visual. No entanto há um maneira bem simples de resolver isso: basta colocarmos um condicional fora do return que manipule o resultado de uma variável com código JSX:

    var flag = Math.random() > 0.5,
      link = null;
    
      if(flag) {
        link = (
          <a
            className="App-link"
            href="https://reactjs.org"
            target="_blank"
            rel="noopener noreferrer"
          >
            Learn React
          </a>
        );
      }

      return (
        {link}
      );

Dessa maneira fica mais fácil enxergarmos o que está acontecendo no código, o que é bom no longo prazo!
