Qual a diferença entre base de dados relacional e não relacional?

Um banco de dados relacional como por exemplo MySQL, PostgreSQL e SQLite 3, são chamado de banco de dados SQL, armazenam dados em tabelas e linhas também chamadas de registros, vinculando informações de várias tabelas por meio do uso de "chaves". Uma chave é um identificador único que pode ser atribuído a uma linha de dados contidos em uma tabela.

Esse identificador exclusivo, chamado de "primary key" ou "chave primária" , pode então ser incluído em um registro localizado em outra tabela quando esse registro tem um relacionamento com o registro primário na tabela principal.

Quando esta chave primária é adicionada a um registro em outra tabela, ela é chamada de "foreign key" ou "chave estrangeira" na tabela associada. A conexão entre a chave primária e a chave estrangeira cria então o relacionamento entre os registros contidos em várias tabelas.

Já os o banco de dados não relacional, ou banco de dados NoSQL, ao contrário do banco de dados relacional, não existem tabelas, linhas, chaves primárias ou chaves estrangeiras. Em vez disso, o banco de dados não relacional usa um modelo de armazenamento otimizado para requisitos específicos do tipo de dados que está sendo armazenado.

Existem quatro tipos não relacionais populares: armazenamento de dados de documentos, banco de dados orientado a colunas, armazenamento de valores-chave e banco de dados de gráficos. Frequentemente, combinações desses tipos são usadas para um único aplicativo.
