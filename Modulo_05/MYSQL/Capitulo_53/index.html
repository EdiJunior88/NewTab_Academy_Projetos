Desafios e Benefícios

Desafios do RDBMS

  • O RDBMS assume uma estrutura de dados bem definida e assume que os dados são amplamente uniformes.
  • Ele precisa que o esquema de seu aplicativo e suas propriedades (colunas, tipos, etc.) sejam definidos antecipadamente antes de construir o aplicativo. Isso não combina bem com as abordagens de desenvolvimento ágil para aplicativos altamente dinâmicos.
  • Conforme os dados começam a ficar maiores, você precisa dimensionar seu banco de dados verticalmente, ou seja, adicionar mais capacidade aos servidores existentes.

Benefícios do NoSQL sobre RDBMS

Os bancos de dados NoSQL sem esquema não definem nenhuma estrutura de dados estrita.

Dinâmico e ágil: Os bancos de dados NoSQL têm boa tendência de crescer dinamicamente com as mudanças de requisitos. Ele pode lidar com dados estruturados, semiestruturados e não estruturados.

Melhor desempenho: Todos os bancos de dados NoSQL afirmam oferecer um desempenho melhor e mais rápido em comparação com as implementações RDBMS tradicionais.

Falando sobre as limitações, como o NoSQL é um conjunto completo de bancos de dados (e não um único banco de dados), as limitações diferem de banco de dados para banco de dados. Alguns desses bancos de dados não oferecem suporte a transações ACID, enquanto alguns deles podem não ter confiabilidade. Mas cada um deles tem seus próprios pontos fortes devido aos quais são adequados para necessidades específicas.
