Bancos de dados NoSQL

Segundo o Wikipédia o NOSQL é um mecanismo para armazenamento e recuperação de dados que são modelados em meios diferentes das relações de tabelas usadas em bancos de dados relacionais, ou seja, NoSQL é um banco de dados não relacional que armazena e acessa dados usando valores-chave. O NoSQL armazena cada item individualmente com uma chave exclusiva, normalmente no formato JSON.

O termo NoSQL foi primeiramente utilizado em 1998 como o nome de um banco de dados relacional de código aberto que não possuía uma interface SQL.

Ao longo de décadas e décadas de desenvolvimento de software, temos utilizado bancos de dados em forma de SQL (Structured Query Language) onde armazenamos nossos dados em tabelas relacionais. No entanto, nos últimos anos, com o tremendo aumento no uso de aplicativos da Internet e da Web 2.0, os bancos de dados cresceram para milhares e milhares de terabytes. Aplicativos como Facebook, Google, Amazon, WhatsApp, etc. deram origem a uma nova era de gerenciamento de banco de dados que segue uma abordagem de design simples, velocidade e escalonamento mais rápido do que os bancos de dados tradicionais. Esses bancos de dados são usados em big data, aplicativos massivos em tempo real e análises.

Como exemplo, considere que você tem um aplicativo de blog que armazena blogs de usuários. Agora, suponha que você precise incorporar alguns novos recursos em seu aplicativo, como usuários que gostam dessas postagens de blog ou comentam sobre elas ou gostam desses comentários. Com uma implementação típica de RDBMS, isso exigirá uma revisão completa do design do banco de dados existente. No entanto, se você usar o NoSQL em tais cenários, poderá modificar facilmente sua estrutura de dados para corresponder a esses requisitos ágeis. Com NoSQL, você pode começar a inserir diretamente esses novos dados em sua estrutura existente, sem criar novas colunas predefinidas ou estrutura predefinida.
