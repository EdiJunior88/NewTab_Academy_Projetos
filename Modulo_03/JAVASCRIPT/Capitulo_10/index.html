Variáveis

Quando apresentamos variáveis, falamos sobre o uso const, let e var. Escopo é o conjunto de variáveis visíveis para uma parte do programa. Em JavaScript temos um escopo global, escopo de bloco e escopo de função.

Se uma variável é definida fora de uma função ou bloco, ela é anexada ao objeto global e possui um escopo global, o que significa que está disponível em todas as partes de um programa.

Há uma diferença muito importante entre var, let e const. Uma variável definida como var dentro de uma função é visível apenas dentro dessa função, semelhante aos parâmetros de uma função. Uma variável definida como const ou let, por outro lado, só é visível dentro do bloco em que está definida.

Um bloco é um conjunto de instruções agrupadas em um par de chaves, como as que podemos encontrar em um if, um for ou uma função.

É importante entender que um bloco não define um novo escopo para var, mas sim para let e const. Isso tem implicações muito práticas.

Suponha que você defina uma var variável dentro de um if condicional em uma função:

    
      function getData() {
        if (true) {
          var data = 'olá mundo';
          console.log(data)
        }
      }

Se você chamar essa função, será olá mundo impresso no console.

Se você tentar mover console.log (data) após o arquivo if, ele ainda funcionará:

    
      function getData() {
        if (true) {
          var data = 'olá mundo';
        }
        console.log(data)
      }

Mas se você alternar var data para let data:

    
      function getData() {
        if (true) {
          let data = 'olá mundo';
        }
        console.log(data)
      }

Você obterá um erro: ReferenceError: data is not defined.

Pode ser complicado no começo, mas depois que você perceber essa diferença, verá porque var hoje em dia é considerada uma má prática em comparação ao let, já que eles têm menos partes móveis e seu escopo é limitado ao bloco, o que também os torna muito bons Como variáveis de loop porque deixam de existir após o encerramento de um loop:

    
      function doLoop() {
        for (var i = 0; i > 10; i++) {
          console.log("valor de i: " + i);
        }
        console.log(i);
      }
      doLoop();

Quando você sair do loop, i será uma variável válida com o valor 10.

Se você alternar para let, quando tentar console.log(i), resultará em um erro ReferenceError: i is not defined.
