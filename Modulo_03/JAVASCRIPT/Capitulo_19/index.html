Teste de mesa

Quando estamos lendo algoritmos, uma das atividades mais comum de fazermos é irmos anotando em um papel ou mentalmente o estado de cada uma das variáveis, esse processo é denominado teste de mesa e é muito útil para identificação de erros em algoritmos. Exemplo:

    
      i = 0
      ENQUANTO i != 11 FACA
          i = i + 2.
          escreva("Ainda não chegamos no onze")
      //Teste de mesa
      /*
      i = 0
      i = 2
      i = 4
      i = 6
      i = 8
      i = 10
      i = 12
      i = 14
      ....
      */

No código acima pelo teste de mesa identificamos que o critério de parada para o loop nunca será atingido, visto que i nunca será igual a 11, resultando em um loop infinito. Podemos corrigir o critério para algo como "i <= 11" para que o mesmo volte a ser atingível.
