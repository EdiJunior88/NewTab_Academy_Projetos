Comentários e funções

Comentários

No código usado para ordenação que vimos há pouco, uma das linhas é a seguinte:

    
      ...
      // Insere cartasSobreAMesa[j] na sequência correta
      ...

Esse “//” indica que essa linha não vai ser interpretada pelo computador, é apenas uma forma de fazer anotações para que outra pessoa que leia seu código consiga entender de forma mais detalhada o que está acontecendo. Os comentários nos códigos são muito importantes e você vai ter provas disso quando estivermos desenvolvendo projetos.

Funções

Muitas vezes um programa é constituído de vários blocos de código que se repetem várias vezes. Imagine que no exemplo anterior, em vez de um baralho possuímos 10 baralhos sobre a mesa, alguns com cartas faltando, o algoritmo deverá então ser executado várias vezes, porém com diretrizes iniciais diferente. Dentro de programação, podemos tratar esse algoritmo com uma função:

    
      Algoritmo ordenarCartas(cartasSobreAMesa)
      inicio
            tamanhoPilha = Tamanho(cartasSobreAMesa)
            PARA j = 2 ATE tamanhoPilha FACA
                carta = cartasSobreAMesa[j]
                // Insere cartasSobreAMesa[j] na sequência correta
        i = j - 1
        ENQUANTO i > 0 E cartasSobreAMesa[i] > carta
          cartasSobreAMesa[i + 1] = cartasSobreAMesa[i]
                      i = i - 1
        cartasSobreAMesa[i + 1] = carta
      fim
      baralhos = [[8,2,4],[6,8,3,2,9],[9,3]]
      i = 0
      ENQUANTO i < Tamanho(baralhos) FACA
        ordenarCartas(baralhos[i])

Dessa forma, a nossa função “Ordenar cartas” vai ser executada no momento que precisarmos de acordo com os parâmetros que passamos, no caso, qual baralho estamos tratando. Dentro da nossa função também indicamos uma função “Tamanho()” que vai contar o número de cartas dentro do baralho.
