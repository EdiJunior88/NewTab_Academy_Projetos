Conceito de lógica de programação

Quando falamos de lógica de programação, entendemos como a forma de pensarmos, os passos que tomamos e as análises que fazemos dentro do desenvolvimento, não nos referimos apenas ao código em si.

Imagine que você tenha um problema, onde você vê uma piscina furada, naturalmente você pensa em formas de resolver a situação, pensando em coisas que provavelmente selariam o furo. Mentalmente evoluímos o problema em mais detalhes, como por exemplo, pensarmos que não basta apenas selarmos o furo, nossa solução precisaria ser à prova d'água.

Quando pensamos em problemas de software, apesar das soluções serem diferentes, a aplicação da lógica para solução ocorre da mesma forma, precisamos evoluir mentalmente o problema e projetar uma solução.

Se temos 20 pedidos com datas diferentes de entrega, a lógica é: anotar as características de cada pedido e a sua respectiva data de entrega e olhar diariamente as anotações para ver se algum pedido deve ser entregue hoje. Isso é o pensamento comum, que não usa nenhum tipo de recurso da informática. Lógica de programação então é quando projetamos um programa como solução, usando ainda este exemplo, para criar um sistema que receba os dados informados pelo usuário e notifique quais pedidos devem ser entregues no dia.
