Link

<link>

Como veremos mais tarde no módulo de CSS, podemos importar estilos para a página de arquivos externos. Isso é muito útil para mantermos nosso código limpinho e fácil de ler, e acredite, você vai querer um código limpinho e fácil de ler. A tag serve para ligar esses (entre outros) arquivos à nossa página. Pense neles como um ctrl+c, ctrl+v que a sua página vai fazer desses arquivos, para daí usar seus recursos.

Para que essa tag funcione certinho vamos usar alguns novos atributos: href, rel e type. Vamos um por um:

  • href: recebe referência ou “caminho” para arquivo que vamos incorporar, isto é, ele vai dizer ao nosso documento onde pegar o recurso externo. Dentro dele vai uma referência feita com protocolo http, então você deve descrever o caminho para a requisição, não a localização “física” do arquivo. Pode parecer confuso falando desse jeito, mas na verdade é só colocar essa string como o endereço de um site que você escreveria no seu browser para acessar um site (https://<seu siteglt.com/><seu arquivo>.css, por exemplo). Se o arquivo que você deseja adicionar estiver no mesmo domínio que seu site, é só colocar o caminho relativo dele em relação à index do site (usando o exemplo anterior, bastaria colocar: <seu arquivo>.css para realizar a importação).
  • rel: estabelece a relação do elemento com a página, isto é, o que ele está fazendo lá dentro.
  • type: diz qual o tipo, isto é, a extensão do arquivo. Os dois últimos não são requisitos (a tag vai continuar funcionando sem elas), mas contribuem no SEO (lembra dele?) da página como um todo.

Ufa, foi bastante coisa, não? Mas tudo bem, com alguns exemplos vai ficar mais fácil de entender, então vamos lá:

<head>
 <title>Exemplo</title>
 <!-- aqui fazemos uma chamada para o domínio local →
 <!-- equivalente a http:///assets/images/favicon.ico -->
 <link rel="icon" href="/assets/images/favicon.ico">

 <!-- aqui fazemos uma chamada para outro domínio -->
 <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@simonwep/pickr/dist/themes/nano.min.css">
</head>

Repare que como essa tag não manipula texto/elementos não precisamos abri-la e fechá-la.
