Elementos do formulário

Muitas vezes precisamos de dados que não somos nós quem geramos, mas que devem vir de quem está utilizando nossa solução. Nesses casos, precisamos nos valer de formulários para coletá-los de nossos usuários.

<form></form>

Com esse elemento declaramos o espaço onde estará nosso form, bem como algumas configurações importantes do seu comportamento que regulamos por dois de seus atributos específicos: action e method.

Especificamente, eles definem como nosso formulário se comunicará com o servidor. Como comentamos antes, não é possível programar, no sentido estrito da palavra, em HTML. Isso quer dizer que toda manipulação, processamento e armazenamento dos dados do formulário deve ser feita fora da página, e esses atributos nos dizem como é realizada essa comunicação. O action define a URL do serviço ou endpoint para onde serão enviados os dados, enquanto method nos diz qual o tipo de chamada que será feito. Lembra do módulo de arquitetura web quando explicamos POST, GET, etc.? São exatamente esses os tipos que definiremos aqui: se enviaremos os dados por POST, GET, PUT, etc.

<input>

Se o <form></form> é o alicerce de nossos formulários, elementos <input> podem ser considerados tijolos: estes são a unidade mais básica de interação com o usuário. Basicamente, eles são as “caixinhas” onde o usuário pode preencher os dados.

Existem diversos tipos de input, de caixas de texto a opções de múltipla escolha, e estes são definidos pelo atributo type. Assim, <input type="text"> representa, como veremos a seguir, um campo de texto simples, para registrar um nome, por exemplo. Abaixo, vamos listar alguns dos tipos mais úteis e usados no dia a dia.

Uma coisa importante de se citar é que, enquanto o HTML suporta diversos tipos de <input>, vamos cobrir apenas os universais ou mais úteis. O HTML5 introduziu vários tipos novos bem legais e convenientes, mas ainda não suportados em todos os navegadores. Caso você queira experimentar com tipos novos, consulte primeiro se os navegadores onde você espera que sua aplicação seja usada dão suporte ao que você deseja usar, usando sites como o Caniuse.

Bom, com isso claro, vamos lá:

  • text: Tipo mais básico de input, e o tipo padrão. É simplesmente uma caixa de texto onde se pode digitar uma linha de texto;
  • submit: Cria um botão que sinaliza para o form enviar os dados. Podemos customizar seu texto interno com o atributo value (mais sobre ele mais tarde).
  • reset: Limpa o formulário, retornando todos os elementos para seu valor padrão. Seu texto também pode ser editado com o value;
  • password: Idêntico ao tipo text, mas oculta o texto digitado, próprio para senhas;
  • hidden: Elemento oculto, para enviar dados para o servidor, mas sem que o usuário saiba. Precisa ter name e value definidos, ou no código ou por Javascript;
  • checkbox: Cria uma caixa de marcação que você pode “ticar” ou não. Como o elemento só tem dois possíveis valores (marcado ou não), é muito bom para valores de sim ou não. Pode receber o atributo checked para vir marcado por padrão;
  • radio: Cria um elemento de uma lista de múltipla escolha. Para definir que elementos radio diferentes pertencem a qual lista, os agrupamos pelo atributo name: elementos com o mesmo valor de name pertencem à mesma lista. Também pode receber o atributo checked;
  • file: Esse tipo é um pouco complicado, mas mesmo assim é bem útil: ele é usado para enviar arquivos para o destino do formulário. Ao clicar nele, é aberta uma janela de seleção de arquivo, que então irá compor o valor do campo. O atributo accept, se presente, recebe uma lista de tipos de arquivos (separados por vírgula) e filtra a janela de seleção para apenas receber arquivos cujo tipo pertença à lista.

Há ainda outros atributos importantes nos inputs:

  • name: É um atributo global, mas que em formulários ganha uma significância ainda maior. O seu valor define o nome do campo ao ser enviado para o servidor. Isso é importante porque, do outro lado, o sistema pode estar esperando valores com um nome determinado que, se estiver errado, pode gerar problemas. Assim, é fundamental prestar atenção aos valores atribuídos a esse atributo;
  • value: Usamos este atributo para definir valores padrões aos campos. Como vimos, nem sempre é ele que é usado para essa função, mas via de regra, se precisamos de um valor pré-definido o usaremos;
  • placeholder: Texto de “exemplo” que aparece no campo quando o mesmo está vazio. Ao contrário do value, esse texto é puramente visual, e não define nenhum dado padrão a ser enviado;
  • maxlength: Define um tamanho máximo de caracteres que podem ser digitados, ignorando valores inseridos após atingir o limite. Aplicável a textareas (veremos mais sobre isso mais tarde) e inputs tipo text;
  • size: Define o comprimento do elemento na tela, em caracteres. Pode ser usado em selects para definir altura, e nesse caso define o tamanho em número de elementos visíveis por vez;
  • readonly: Elementos com esse atributo são mostrados na tela, e se tiverem um valor configurado pelo atributo value enviam dados junto com o resto do formulário, mas o usuário não pode interagir com ele. Mais útil para informar valores dinâmicos manipulados via JS;
  • disabled: Similar ao anterior, com a diferença de que os dados contidos nele não são enviados;
  • autofocus: Se presente, automaticamente dá foco ao elemento, eliminando a necessidade de clicar no mesmo para editar seu valor quando o form é iniciado.

<select></select>

O elemento input é uma ferramenta poderosa e flexível na construção de formulários. No entanto, ele não é o único, ou necessariamente melhor modo de coletar dados. Pense no tipo radio, por exemplo: se tivermos algumas poucas opções é uma opção excelente para uma caixa de seleção, mas à medida que as opções crescem, o conjunto ocupa mais espaço de tela, potencialmente comprometendo o layout e a legibilidade do resto do formulário. Uma forma como podemos resolver isso é com o elemento <select></select>.

Ele renderiza um lista suspensa com diversas opções, que se fecha após a seleção de um valor. As opções são definidas usando a tag <option></option>, que envolve o texto que aparecerá na lista para o usuário. Essa tag é que recebe o atributo value, que define o valor que será atribuído à lista quando selecionada. Para definir um valor padrão então podemos usar o atributo selected na opção cujo valor desejamos que seja o inicialmente selecionado.

Além disso, podemos ainda usar o <optgroup></optgroup> para agrupar opções em categorias para melhor visualizá-las. Basta colocar opções da mesma categoria dentro dela e usar o atributo label para definir qual o nome da categoria. Note que quando esse nome aparece na lista não conseguimos selecioná-lo: ele serve apenas para dividir os elementos visualmente.

Um exemplo simples:

<select id="vegetais">
  <optgroup label="Frutas">
   <option value="a">Banana</option>
   <option value="b" selected>Laranja</option>
   <option value="c">Manga</option>
</optgroup>

  <optgroup label="Legumes">
   <option value="d">Brócolis</option>
   <option value="e" selected>Chuchu</option>
   <option value="f">Cenoura</option>
  </optgroup>
</select>

<textarea></textarea>

Com o tipo text já vimos como o usuário pode escrever uma linha de texto. Mas se for necessário escrever um bloco muito grande rapidamente escrever naquela uma linha se tornará em uma tarefa árdua, com o usuário tendo pouco espaço para ir e voltar dentro do que ele escreveu. É aqui que o <textarea></textarea> entra. Ele define um espaço para entrada de texto, mas com múltiplas linhas, de modo que é mais fácil escrever trechos mais extensos, como mensagens ou recados.

Ao invés de receber um atributo value para definir um valor pré definido, este último é o texto que envolvemos com a tag. Também pode receber valores em cols e rows, ditando o número de colunas e linhas visíveis respectivamente. Exemplo:

<textarea id="lipsum" cols="40" rows="10">Lorem ipsum dolor sit amet...</textarea>

<label></label>

Vimos bastante coisa nesse módulo, não é? Mas calma, tem mais uma coisinha. Se apenas irmos jogando os elementos de nosso formulário, nossos usuários não terão nenhuma referência do que aqueles campos representam. Uma solução rápida é simplesmente adicionar texto ao lado ou em cima de cada elemento. No entanto, há uma solução superior: a tag <label></label>. Com ela podemos ligar um trecho do nosso documento envolto por ela a um elemento do formulário.

Isso é feito como atributo for, que recebe o id do input, select, ou textarea que queremos descrever. A vantagem dessa abordagem é que, além de tornar o código mais descritivo e organizado, podemos selecionar o elemento clicando no conteúdo da label, o que tem muitas aplicações, principalmente ao lidar com checkboxes e radios.

Nome: <input type="text" id="nome" name="nome" placeholder="Nome">

vs.

<label for="labelNome">Nome:</label>
<input type="text" id="labelNome" name="labelNome" placeholder="Nome">
